## Fact
A clock trained on a reference cohort will give **systematically wrong** ages on a
new dataset generated by a different lab/protocol/platform, because technical batch
effects shift the feature distribution the model was fit on. The standard fix is to
**batch-correct the reference and query together before prediction**, treating the
reference as the anchor batch. For RNA-seq counts, **ComBat-seq**
(`inmoose.pycombat_seq`) does this on raw counts under a negative-binomial model;
classic **ComBat** does it on log-transformed/continuous data.

## Why it matters
Skipping this step is a common, silent failure: predictions look plausible but are
offset or compressed by a batch-specific bias, and "age acceleration" between query
groups becomes uninterpretable. Correcting jointly with the reference as reference
batch preserves the scale the clock expects.

## Caveats
Batch correction assumes batch and the biological signal (age, condition) are not
confounded — if a batch **is** a condition, correction removes the effect you want.
Provide known biological covariates to protect them. ComBat-seq needs raw counts
([[count-models-need-raw-counts]]); do not feed it already-normalized data. Over-
correction can erase real cross-cohort differences.
